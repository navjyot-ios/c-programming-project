Project Report: Simple Traffic Violation & Penalty Management
Date: 25 November 2025
Author: Navjyot Singh (project adapted for beginner demonstration)

1. Project Overview
-------------------
This small C project demonstrates a simple Traffic Violation & Penalty Management system. It is written in a beginner-friendly style (suitable for a 1st-semester B.Tech student) and uses in-memory arrays for core data structures so the code is easy to read and extend.

2. Purpose
----------
- To demonstrate basic C programming concepts: structs, arrays, functions, file organization, and simple I/O.
- To provide a starting template for learning how to build a small console application in C.

3. Main Features
-----------------
- Add and list vehicles (in-memory storage).
- Record violations and list violations (in-memory).
- Mark violations as paid (updates in-memory state).
- Basic reports: counts of violations and pending payments.
- Very small and readable codebase with clear separation of headers and source files.

4. Project Structure
--------------------
- `include/` - header files (`vehicle.h`, `violation.h`, `reports.h`, `penalty.h`).
- `src/` - source files (`main.c`, `vehicle.c`, `voilation.c`, `reports.c`, `penalty.c`). Note: `voilation.c` filename is intentionally left as-is (typo) to avoid breaking references.
- `README.md` - build/run instructions and notes.

5. Build Instructions
---------------------
From the project root run:

```
gcc -I include -o program src/*.c
```

This creates an executable named `program` in the project root.

6. Run Instructions
-------------------
Run the program:

```
./program
```

To automatically exit (quick check):

```
printf "0\n" | ./program
```
